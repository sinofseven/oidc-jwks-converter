# リリースバージョン決定

## 目的
新しいバージョンのタグを打つにあたって、マイナーバージョンとパッチバージョンどっちを上げるべきかの判断材料にしたい

## 要望
v0.1.0から現在のコミットまでの差分は何があるのか概要を教えて欲しい

## プラン

### バージョン決定の判断基準
- Semantic Versioning（セマンティックバージョニング）に基づいた判断
- コア機能の変更：なし
- ユーザー向け新機能：あり（Homebrew・crates.io での配布方法拡張）
- 品質・信頼性：向上（ビルドパイプライン改善、ドキュメント充実）

### v0.1.0 からの主要な変更

1. **配布方法の拡張**（新機能級）
   - Homebrew Formula 公開対応：`publish_formula.yml` ワークフロー、`about.md` 作成
   - crates.io 公開準備完了：`Cargo.toml` メタデータ、ドキュメントコメント追加
   - README に複数のインストール方法を記載

2. **ビルド・CI パイプラインの大規模改善**
   - Rust キャッシュ機能追加（ビルド時間削減）
   - Linux ARM 環境の安定化（rust-musl-cross 導入）
   - 複数プラットフォーム（Windows、macOS、Linux）での確認

3. **依存関係の更新**
   - `reqwest` 0.11 → 0.13、`thiserror` 1.x → 2.x（メジャーアップデート）

4. **ドキュメント・メタデータの充実**
   - `.gitignore` に `.DS_Store` 追加
   - `about.toml` にライセンス追加

### 推奨結論
**次のバージョンタグ：v0.2.0（マイナー版）**

**理由：**
- ユーザー向けの配布方法が大幅に拡張（Homebrew・crates.io）
- ビルドパイプラインと品質が向上
- コア機能に変更はなく、破壊的変更もない
- Semantic Versioning では、新機能追加時はマイナー版を上げる
