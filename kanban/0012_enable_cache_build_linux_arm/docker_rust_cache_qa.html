Docker Rustビルドキャッシュ Q&A

質問

以下のDockerコマンドにキャッシュを効かせることはできますか？Rustの依存関係周りなど。

docker run --rm \
  -v /home/runner/work/oidc-jwks-converter/oidc-jwks-converter:/workspace \
  -w /workspace \
  ghcr.io/rust-cross/rust-musl-cross:arm-unknown-linux-musleabihf \
  cargo build --release --target arm-unknown-linux-musleabihf
回答

RustのビルドキャッシュをDockerで効かせるには、主に2つのアプローチがあります。

1. GitHub Actions cache + ボリュームマウント シンプル・推奨

~/.cargo/registry と target/ をキャッシュする方法です。

- name: Cache cargo registry
  uses: actions/cache@v4
  with:
    path: |
      ~/.cargo/registry
      ~/.cargo/git
    key: ${{ runner.os }}-cargo-registry-${{ hashFiles('**/Cargo.lock') }}
    restore-keys: |
      ${{ runner.os }}-cargo-registry-

- name: Cache build artifacts
  uses: actions/cache@v4
  with:
    path: target/arm-unknown-linux-musleabihf
    key: ${{ runner.os }}-musl-arm-${{ hashFiles('**/Cargo.lock') }}-${{ hashFiles('src/**/*.rs') }}
    restore-keys: |
      ${{ runner.os }}-musl-arm-${{ hashFiles('**/Cargo.lock') }}-
      ${{ runner.os }}-musl-arm-

- name: Build
  run: |
    docker run --rm \
      -v /home/runner/work/oidc-jwks-converter/oidc-jwks-converter:/workspace \
      -v $HOME/.cargo/registry:/root/.cargo/registry \
      -v $HOME/.cargo/git:/root/.cargo/git \
      -w /workspace \
      ghcr.io/rust-cross/rust-musl-cross:arm-unknown-linux-musleabihf \
      cargo build --release --target arm-unknown-linux-musleabihf

ポイント

  • ~/.cargo/registry をホスト側でキャッシュしてコンテナにマウント → crates.ioからの再ダウンロードを防ぐ
  • target/ ディレクトリもキャッシュ → インクリメンタルビルドが効く
  • target/ はサイズが大きくなりがちなので注意

2. sccache を使う より本格的

- name: Install sccache
  uses: mozilla-actions/sccache-action@v0.0.9

- name: Build
  env:
    SCCACHE_GHA_ENABLED: "true"
    RUSTC_WRAPPER: sccache
  run: |
    docker run --rm \
      -v /home/runner/work/oidc-jwks-converter/oidc-jwks-converter:/workspace \
      -v $HOME/.cargo/registry:/root/.cargo/registry \
      -v $HOME/.cargo/git:/root/.cargo/git \
      -e SCCACHE_GHA_ENABLED=true \
      -e RUSTC_WRAPPER=/usr/local/bin/sccache \
      -v $(which sccache):/usr/local/bin/sccache \
      -w /workspace \
      ghcr.io/rust-cross/rust-musl-cross:arm-unknown-linux-musleabihf \
      cargo build --release --target arm-unknown-linux-musleabihf

ただし sccache はコンテナ内のバイナリ互換性（musl環境）に注意が必要で、設定が複雑になりがちです。

キャッシュ対象の比較

キャッシュ対象効果サイズ
~/.cargo/registry依存クレートの再ダウンロード防止中（数百MB）
~/.cargo/gitgit依存クレートのキャッシュ小
target/インクリメンタルビルド大（1GB超も）

クロスコンパイル用途ではアプローチ1のシンプルな方が安定しています。target/ のキャッシュは初回保存に時間がかかるため、Cargo.lock のハッシュをキーに含めて依存更新時は破棄するのが定石です。
