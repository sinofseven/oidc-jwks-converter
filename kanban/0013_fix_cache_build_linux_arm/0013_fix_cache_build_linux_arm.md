# build-linux-armのキャッシュ機構修正

## 目的

キャッシュ機構をいれてから何度か動かしているが、キャッシュが効いていないように見える。ビルド高速化するためになんとかして欲しい。

## 要望

build-linux-armにキャッシュ機構がうまく動いていない。修正して欲しい。

## 備考

CIのビルドログは `ci.log` と言う名前でタスクファイルと同じディレクトリに保存している。

## プラン

build-linux-arm ジョブのキャッシュが効率的に機能していない根本原因は、**Docker 権限ミスマッチ**と**キャッシュキーの不備**です。

### 修正内容

1. **Docker UID マッピング追加**（最優先）
   - docker run コマンドに `--user 1001:1001` を追加
   - ホストの runner ユーザー（UID 1001）と Docker コンテナ内の実行ユーザーを同一化
   - キャッシュ復元後、cargo がファイル権限エラーなく更新可能に

2. **キャッシュキーの改善**
   - registry キー：`${{ runner.os }}-cargo-registry-arm-...` に "arm" プレフィックス追加
   - build artifacts キー：`{{ hashFiles('src/**/*.rs') }}` ハッシュを削除（Cargo.lock のみに単純化）
   - 他の Linux ジョブとのキー重複防止、ヒット率向上

3. **長期改善案**（別タスク）
   - 将来的には Docker 外でのネイティブビルド + Swatinem/rust-cache への移行を検討
   - 現在は短期修正で対応、Swatinem は Docker 環境未対応のため推奨しない

### 期待効果

- ビルド時間短縮（目標：85秒以下）
- インクリメンタルビルドの正常化
- キャッシュヒット率向上
